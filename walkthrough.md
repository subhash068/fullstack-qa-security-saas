## 1. Database Architecture & Audit Telemetry

The database has been fully redesigned to conform to Third Normal Form (3NF).
* **Database Schema DDL**: [schema.sql](file:///c:/Users/windows-11/Desktop/QA_project%201/database/schema.sql)
* **Seed SQL Script**: [seed.sql](file:///c:/Users/windows-11/Desktop/QA_project%201/database/seed.sql)

### Table & Separation Roles:
1. `roles` (RBAC definitions) & `permissions` (access rules) linked via junction table `role_permissions` for modular permission mapping.
2. `users` (credentials & login credentials) separated in 1:1 format from `profiles` (bio, phone, date of birth, avatar URL) to guarantee high-frequency indices are kept small and fast.
3. `user_sessions` (token tracking, device signatures) & `password_resets` (recovery tokens).
4. `audit_logs` (application logs) & `audit_trail` (database DML trigger log).

### Row-Level Change Auditing:
A PL/pgSQL trigger function automatically captures changes post-transaction (`AFTER INSERT OR UPDATE OR DELETE`) on `users`, `profiles`, `roles`, and `permissions` tables.
* It logs state changes into `audit_trail` as `JSONB` objects.
* **Security Safeguard**: The trigger scans the data and automatically strips all occurrences of `password_hash` fields from both old and new states before they hit the audit records.

## 2. Refactored Service Layer Backend

The backend has been refactored to extract business logic from API routers and repositories into a structured **Service Layer**:
* **`AuthService`**: Handles security tasks like login, registration, token refreshes, and recovery workflows.
* **`UserService`**: Handles profile adjustments, credentials updates, and administrative CRUD operations.
* **`AdminService`**: Aggregates statistics and collects forensic audit logs.

## 4. Server-Side Protection (Edge Router Proxy)

We implemented an Edge Router Proxy mapping (`frontend/proxy.ts`) which validates sessions on the server:
* Restricts access to `/dashboard`, `/profile`, and `/admin` paths to authenticated users possessing valid session cookies.
* Restricts access to administrative sub-modules (`/admin/**`) exclusively to users flagged with the `Admin` role. Redirects standard users to a 403 Forbidden page.
* Blocks authenticated users from returning to login/registration forms (redirects them back to `/dashboard`).
